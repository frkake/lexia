# CEFR 語彙帯域アセット (`cefr-bands.json`)

パッセージ生成の語彙難易度ゲート（B-4）が使う、レンマ → CEFR 帯域の静的辞書です。
`cefrDictionary.ts` がビルド時にこの JSON を取り込み、`PassageValidator` のオフバンド率検査へ注入されます。

## 出典とライセンス

- **出典**: 一般的な英語頻度・教育的知見にもとづき Lexia チームが独自に手作業でキュレーションした頻度階層リストです。
- **重要**: CEFR-J Wordlist・Oxford 3000/5000・その他ライセンス付きコーパスから派生・複製したものではありません。したがって再配布上の制約はありません。
- **ライセンス**: このデータ（`cefr-bands.json`）はパブリックドメインに献呈します（**CC0 1.0**）。用途・改変・再配布は自由です。

## 帯域モデル

- アプリの `Cefr` 型は 5 帯域（`A2 B1 B2 C1 C2`）。真の A1 コアは最下位バケットである **A2 に畳み込み**ます（バリデータが比較できる最下位帯域）。
- 語が複数バンクに現れた場合は**下位帯域が優先**されます（同綴異義語の最も一般的な読みを上位帯域として誤検出しないため）。
- リスト外の語（固有名詞・不規則活用・希少派生など）は `undefined` を返し、バリデータが意図的にスキップします（従来からの「不明語はカウントしない」寛容設計）。

## レンマ化（`cefrDictionary.ts` 側）

パッセージは表層形トークン（`negotiated`, `terms` など）を持つため、ルックアップは
「表層形 → 規則的な脱活用候補列」を順に試し、辞書に存在する最初の候補の帯域を返します。
複数形/三単現・過去/過去分詞・現在分詞・比較級/最上級・副詞 `-ly`・語末サイレント `e`・
子音重複（`running→run`）を扱います。不規則活用は辞書に載っていなければスキップされます。

## 再生成

語を追加・修正するときは生成スクリプトのバンク（`A2`〜`C2`）を編集し、再生成します。
JSON を直接編集しないでください（スクリプトが重複除去・下位帯域優先・ソートを担保します）。

```
node scripts/gen-cefr-bands.mjs
```

現状: 約 3,600 レンマ / raw ~36KB / **gzip ~14KB**（受け入れ基準 gzip ≤ 150KB を満たす）。
