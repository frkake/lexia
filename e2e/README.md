# E2E / ビジュアル回帰テスト運用

Playwright による E2E とビジュアル回帰(visual regression)の実行方法と、ベースライン画像の運用ルールをまとめる。

## 実行方法

```sh
pnpm test:e2e            # 全 spec を実行(スナップショットは既存ベースラインと比較)
pnpm test:e2e:update     # スナップショットを撮り直してベースラインを更新(下記ルール厳守)
```

- 設定: `playwright.config.ts`。プロジェクトは `desktop-chromium`(1280×900)と `mobile-webkit`(414×842)の 2 つ。`webServer` は `vite --port 5173`。
- スナップショット比較の許容差は `toHaveScreenshot.maxDiffPixelRatio = 0.02`、アニメーションは無効化。
- `/api/*` は `e2e/helpers.ts` の `mockApi` で決定的にモックする(実 LLM/画像 API は叩かない)。

## ビジュアル回帰の仕組み

- 撮影対象は `gallery.html`(`src/gallery.tsx` + `src/gallery.fixtures.ts`)の決定的フィクスチャ。URL ハッシュごとに 1 プレゼンテーショナル画面をレンダリングする。
- 対象画面(`e2e/visual.spec.ts` の `SCREENS`): `dashboard` / `reading` / `reading-grid` / `wordcard` / `review` / `setup` / `wordbook`。
- ベースラインは `e2e/visual.spec.ts-snapshots/` に `<画面名>-<プロジェクト>-<プラットフォーム>.png`(例: `reading-desktop-chromium-darwin.png`)で保存される。フォントラスタライズが環境で異なるため **darwin / linux の両プラットフォーム分**を保持する。
- ピクセル安定性に加えて、`visual.spec.ts` は習熟度色・注釈エンコード・注釈カテゴリのチップ色といった **load-bearing なデザイントークン**を computed style で明示的にアサートする。

## ベースライン運用ルール(R6・最重要)

### 崩れが写った画像を基準にしない

`--update-snapshots` は「**意図した見た目の変更を新しい正基準として確定する**」操作であって、レイアウト崩れ・要素欠落・フォント未適用・文字化けを基準に焼き付ける操作ではない。崩れたスクリーンショットをベースライン化すると、以後その画面では**同種の崩れを二度と検知できなくなる**(回帰の見逃し)。

したがって更新は必ず次の手順で行う。

1. UI 変更を**完成**させる(途中の崩れた中間状態で撮らない)。
2. ローカルで `pnpm test:e2e` を実行し、失敗した画面の差分を確認する。
3. 生成された各 `<画面名>.png`(および Playwright の diff 画像)を**目視**し、差分が意図どおりの変更のみであることを確認する。意図しない崩れが 1 つでもあれば、更新せずに UI を直してから 2 に戻る。
4. 問題なければ `pnpm test:e2e:update` でベースラインを更新する。
5. 更新後の PNG を画像ビューア / `git diff --stat` で再確認してからコミットする。

### プラットフォームをまたいで上書きしない

darwin と linux ではフォントのラスタライズが異なるため、**撮影環境と同一プラットフォームのベースラインだけ**を更新する。他プラットフォーム分は該当環境で撮り直すこと。ローカル(darwin)で撮ったものを linux ベースラインとして上書きしてはならない(逆も同様)。

### 多重更新は各波の最後に 1 回へ集約する(R6)

フォント導入 → レイアウト改修 → コントラスト調整のように見た目の変更が連続する場合、変更のたびに撮り直すと最低 3 回ベースラインが差し替わる。各波の**末尾で 1 回だけ**再生成し、途中の崩れた中間状態を基準化しない。

## CI

現状 CI ワークフローは未整備。CI 導入時はビジュアル回帰を**必須ジョブ**にし、撮影環境(ランナー OS)を固定して `<プラットフォーム>` を正基準とする。それまでは UI に触れる変更は、手元で `pnpm test:e2e` を通してからマージする。
